完全削除（CompleteDelete）とは

推奨動作環境

Windows 95/98/Me/NT4/2000/XP/Vista
FATファイル・システム （推奨）
NTFS（NTファイルシステム） (問題点はこちらを参照)
特別なファイル保護機能を組み込んでいないこと
削除するファイルの上書き、消去が出来る環境であること

MFC42.DLL および MSVCRT.DLL がシステムフォルダに存在すること（Visual C++ 6.0 対応版） 古いバージョンのDLLの場合、起動できません。

このプログラムには、日本語と英語の２種類の設定が含まれています。日本語以外のWindowsで起動した場合、自動的に英語モードで動作します。（表示も内部処理も）

DOD 5220.22-M 対応について

アメリカ国防総省 国家産業セキュリティ適用マニュアル (NISPOM) における Automated Information System Security の Clearing and Sanitization に関しては現在のところ、対応していません。

このソフトの特徴

ファイル毎に完全削除ができる
Windows標準の操作性と、表示方法を導入し違和感がない
NT/2000/XP/Vista において、アドミニストレータ権限を必要としない

完全削除について

「間違って削除したファイルを復活することができる」
一見便利そうなこの言葉の裏側に、大きな危険が潜んでいます。
普通に削除しただけのファイルは、容易に復活できることもあります。

Windows95/98/MeはMS-DOSのファイルシステム(FATファイルシステム)をベースに拡張されており、セキュリティーに対する考え方もたいそう低いものとなっています。抹殺してしまいたい「情報」を削除する場合、Windows標準の「削除」処理では不十分です。
また、Windows NT/2000/XP/Vista で利用されているNTFS（NTファイルシステム）も、NTにログオンせずに、DOS上から読み出すソフトウエアは存在していますし、暗号化されていないデータはディスク自体の解析に対してはまったく無防備です。

そこで、｢完全削除(CompleteDel)｣を使用することにより、より完全な形でデータファイルを削除することができます。
ファイルの削除と“完全”削除の違いはこちらをご覧ください。

なお、機密度の相当高いファイルを削除する場合、データ漏洩阻止の保証はできませんのでご注意ください。業務用には、専用ソフト等の使用をお勧めします。
また、機密情報の完全削除をより完璧に行う場合は、この｢完全削除(CompleteDelete)｣使用後にWindowsの｢デフラグ｣を実行する、空き領域を完全消去するプログラムを実行するなど、ディスク内開放領域の撹乱も同時に行う事をお勧めします。

空き領域の削除 （Windows標準の機能）

Windows 2000/XP/Vista は標準で空き領域の削除コマンドを実装しています。
コマンド・コンソール（cmd.exe）を開き、cipher.exe を実行して空き領域を完全削除することが出来ます。

【Windows で Cipher.exe を使用して削除済みのデータを上書きする方法 ： http://support.microsoft.com/kb/315672/ja 】

完全削除の方法

エクスプローラで削除したいファイル(複数可)を選択し、｢完全削除｣アイコンにドラッグ＆ドロップする。または、右クリック･メニューの｢送る｣を実行する。

完全削除の方法関する詳細情報はこちらをご覧ください。

完全削除が難しいファイルの例

このプログラムは標準的な構成のWindowsワークステーションを対象としており、ファイル保護装置またはソフトウエアが付加された環境下では「完全削除」を行えないことがあります。

例 ：
ハードウエア的にファイルシステムの多重化をしている場合（ＨＤＤのミラーリングなど）
ファイル保護ソフトが働いている場合（フォール･トレラント･ドライバ等）
ネットワーク･ファイル･システム上のファイル

次のファイルは、完全削除できません

上書きできないメディア上のファイル（ＤＶＤ−ＲＯＭ、ＤＶＤ±Ｒ・ＲＷ、ＣＤ−ＲＯＭ、ＣＤ−Ｒ・ＲＷなど）
